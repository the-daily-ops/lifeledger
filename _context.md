### 1. NEVER rewrite lifeledger-viewer.html manually
The viewer is auto-generated by `generate-viewer.py`. It reads all .md records and builds the full HTML. **Do not read, rewrite, or update lifeledger-viewer.html directly** — it costs ~25,000 tokens and is error-prone. Instead tell the user to run:

### 2. NEVER rewrite dashboard.html manually
The dashboard is auto-generated by `generate-dashboard.py`. Tell the user to run:
```
dashboard
```
or the full path:
```
python3 "/Users/coto/Library/Mobile Documents/com~apple~CloudDocs/LifeLedger/generate-dashboard.py"
```

### 3. Claude's job for these files is ONLY:
- Write/update individual `.md` record files in LifeLedger folders (small, cheap)
- Write/update the Obsidian vault task file
- Then tell the user to run `dashboard` or `viewer` to regenerate

### 4. NEVER overwrite the vault file carelessly
The main task file (`2026-02-25.md`) is the most important file in the system. Always read it fully before writing. Use targeted edits — never write the whole file unless restoring from a known good state. Git backup runs automatically via the watcher.

## Version Control & Backup

Three backup layers always running:
1. **Vault git** — commits every watcher cycle (5 sec) to local git in vault folder
2. **LifeLedger GitHub** — pushes all records on every `dashboard` run
3. **iCloud** — continuous sync to Apple servers

**To restore vault after accidental overwrite:**
```bash
cd "/Users/coto/Library/Mobile Documents/iCloud~md~obsidian/Documents/Digital Bullet Journal"
git log --oneline
git checkout HEAD~1 -- "01 Daily/2026-02-25.md"
```
